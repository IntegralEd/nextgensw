Announcement control

Edit the live announcement (top banner + first-visit modal). Changes publish to the public site when you click Publish to live site; the rebuild takes about 45 seconds.

Pulls the current Status = Published record into nextgensw.org. Bump Version before publishing if you want returning visitors to see the new announcement even if they dismissed the previous one.

How this works (and what each field controls)
  1. Edit the row below. Status = Published is what goes live; Draft and Archived are ignored.
  2. For a brand-new announcement, bump Version by 1. The site uses Version as a dismissal key, so a returning visitor who dismissed v1 will see v2 again.
  3. Click Publish to live site. Watch this toast for confirmation, then refresh nextgensw.org in about a minute.
  4. After the deadline passes (the day after Deadline_Date), the Active_Announcements view drops the record automatically and the banner disappears next time the site rebuilds. No manual cleanup needed.

Field cheat sheet

Visible
Master kill switch. Off = nothing renders on the public site, regardless of the two flags below.
Show_Banner
Top strip across every page. Dismissible by visitors; re-appears when Version bumps.
Show_Modal
First-visit popup. Once per visitor per Version.
Show_Countdown
Off hides the "N days left" chip / pill on both placements.
Preset
V1 supports Application Deadline. Add Funding Match / Info Session as a new singleSelect option when those announcement types are needed.
Deadline_Date
Date only. Treated as end-of-day America/New_York. Drives the countdown and the auto-archive behavior.
Anchor
Where the CTA button goes. Accepts a page anchor (#apply) or a full URL.
Starts_At · Ends_At
Optional. Use to stage the next deadline in advance — site won't render the announcement before Starts_At or after Ends_At.
Version
Manual integer. Bump when re-surfacing to returning visitors.
