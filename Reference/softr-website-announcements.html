Announcement control

This page edits the message that appears at the top of nextgensw.org (the strip across every page, and the popup that appears on first visit). Changes go live when you click the red button below.

After editing the row below, click this. The site rebuilds in about 45 seconds. You'll see a notification right below this box letting you know it worked.

How to use this page (open for instructions)

Step by step

  1. Find the row you want to edit in the table below. Status is what controls whether a row is live: only rows set to Published appear on the public site.
  2. Edit any field you want to change.
  3. If you want returning visitors to see the new message even if they already dismissed the previous one, increase the Version number by 1 (so if it's at 1, make it 2). Skip this step if you're just fixing a small typo.
  4. Click the red Publish to live site button at the top of this page.
  5. Wait for the green confirmation message to appear below the button. The public site will reflect your edits in about 45 seconds.
  6. When the Deadline_Date passes (or, for non-deadline announcements, the Ends_At date), the announcement disappears from the site automatically — you don't need to come back and change anything.

What each field does

Name
An internal label for you and the team. Visitors never see this. Use something memorable like "July 15 priority deadline".
Status
Draft = still working on it, not live. Published = live on the site. Archived = old, kept for history but not live.
Preset
What kind of announcement this is. Right now there's only one option (Application Deadline); we'll add others when we want to publish different kinds of announcements (e.g., a donation matching event or an info session).
Eyebrow
The small line of bold all-caps text above the main message. In the current banner it reads "PRIORITY DEADLINE". Keep it short (one to three words).
Headline
The main message. Keep it short enough that it fits on one line on a desktop browser (around 48 characters).
Subtext
Longer explanation. Only shown in the first-visit popup, not on the strip across the top.
Button_Text
The words shown on the button (e.g., "Apply now").
Anchor
Where the button takes the visitor when clicked. See the next section below for how to get one.
Deadline_Date
The date the announcement counts down to. Treated as end-of-day Eastern Time. For an application deadline this is the field that drives the "N days left" countdown and the auto-hide after the date passes. For non-deadline announcements (e.g., an info session), you can leave this blank and use Ends_At instead.
Show_Countdown
Turn this on to show the "N days left" chip next to the headline. Turn off to hide it (useful for non-deadline announcements).
Show_Banner
Turn on to show the strip across the top of every page. Turn off if you only want the popup.
Show_Modal
Turn on to show the popup on the first visit of the day. Turn off if you only want the strip.
Visible
Master on/off switch for the whole announcement. Turn this off and nothing appears on the public site, no matter what the other settings say. Use this for an emergency take-everything-down.
Starts_At
Optional. The earliest moment the announcement should appear on the site. Leave blank and the announcement appears as soon as you publish it. Set a future date/time to stage the next announcement ahead of time — it will sit hidden until that moment arrives. (You can edit and Publish a future-dated announcement now; the site won't show it until Starts_At.)
Ends_At
Optional. The latest moment the announcement should appear. Use this for announcements that aren't tied to an application deadline (e.g., an info session or limited-time fundraising match). If Deadline_Date is blank, Ends_At is what keeps the announcement alive — once it passes, the announcement auto-hides.
Version
A whole number. Each time you want returning visitors to see a new announcement (even if they dismissed the old one), increase this by 1. Behind the scenes the site uses Version to tell whether a visitor has seen this particular announcement before.

How to get an Anchor

The Anchor tells the site where to send the visitor when they click the button. You have three options:

  • A spot on the homepage: open nextgensw.org in a new tab. In the navigation at the top, click one of the section links (Why, How, Timeline, Team, Get Involved). The page jumps to that section and the URL in your browser changes to something like https://nextgensw.org/#why. Copy that whole URL — or just the part starting with # (e.g., #why) — and paste it into the Anchor field.
  • The Apply button shortcut: if you want the button to scroll to the Apply section, the easiest value is just #apply.
  • An external link: if the button should open something outside the site (a Google Form, a donation page, etc.), paste the full URL including https://.

If something goes wrong

If the red button is grey and won't click, give it a few seconds and try again. If the notification message that appears below the button is red instead of green, there was a network problem; reload this page and try once more. If a publish goes through but the public site doesn't update after a minute or two, ask David to check the Netlify deploy log.
